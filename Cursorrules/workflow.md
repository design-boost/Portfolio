# 開発フローのルール

1. **作業前のバックアップ**
   - `git status` でクリーンか確認。
   - クリーンでなければ `git add` → `git commit` で現状を保存。
   - 可能であればリモートに `git push` しておく。

2. **作業ブランチの作成**
   - `git checkout -b <feature-branch>` でブランチを切り、そこで修正を行う。

3. **修正と確認**
   - コードを書き換え、`npm run dev` などでローカル確認。
   - 必要に応じてテストを実行。

4. **バックアップとして再コミット/プッシュ**
   - 作業が落ち着いたら `git add` → `git commit`。
   - リモートがある場合は `git push`。

5. **その他**
   - ルールを変更したい場合はこのファイルを更新する。
